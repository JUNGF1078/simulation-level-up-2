03 · Architecture

Struktur layout HTML

<body> <img id="bg-desk"> ← background bebas stretch <div id="app-wrapper"> ← fixed overlay z-index:10 <img id="logo"> ← logo pojok kiri <div id="phase-bar"> ← progress steps <div id="budget-oval"> ← sisa budget <div id="screens-wrap"> ← konten utama ── #briefing ── #planning ├─ .decisions-panel (scroll) └─ .preview-panel (timer) ── #simulation ── #results ── #laporan

🎛 CSS Variables Config

Semua posisi dikontrol via :root variables. Bisa di-tune dari browser pakai Live Config Panel tanpa edit code.

📍 --pb-top — phase bar vertical

📍 --budget-right — budget oval

📍 --nb-left/right — panel lebar

📍 --pv-top/right — timer panel

🔑 Key Principle

Background stretch bebas (cover), UI content posisi fixed — tidak terdistorsi di resolusi berbeda

07 · Dev Tooling

Live Config Panel

Problem: Posisi elemen (phase bar, budget oval, panel) berbeda di setiap layar & resolusi. Edit CSS → refresh → lihat hasilnya → edit lagi = lambat.

Solution: Floating panel di browser dengan sliders yang langsung mengubah CSS variables secara realtime.

🔄 Workflow

1

Buka localhost:3001

2

Drag sliders sampai posisi pas

3

Klik "Copy CSS"

4

Paste ke styles.css

5

git push ke Railway

📐 Variables yang bisa di-tune

Phase Bar

top, shift-X, step spacing

Budget Oval

top, right (0–25vw)

Left Panel

top, bottom, left, right

Timer Panel

top, right, width

💡 Insight

Pisahkan design config dari logic. Posisi bukan urusan game.js — semua di CSS vars sehingga bisa diubah tanpa menyentuh logic game.

08 · Deployment

Pipeline dari dev ke prod

Dev localhost:3001 ↓ Adjust via Live Config Panel ↓ Copy CSS → paste ke styles.css ↓ git add assets/css/styles.css git commit -m "chore: adjust layout" ↓ git push origin2 main ↓ Railway (auto-deploy dari GitHub) ↓ npm start → serve . -l $PORT ↓ Production URL ✓

📦 package.json

{ "scripts": { "start": "serve . -s -l ${PORT:-3000}" }, "dependencies": { "serve": "^14.2.1" } }
